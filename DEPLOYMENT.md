# Deployment Guide — Next Chapter Travel

## Architecture

```
Browser
  │
  ├── Static assets (HTML/JS/CSS/videos)
  │     └── Netlify CDN  ←── GitHub main (auto-deploy)
  │
  └── API calls (/api/*)
        └── Render Web Service  ←── GitHub main (auto-deploy)
              └── MySQL/TiDB (PlanetScale, TiDB Cloud, or Render MySQL)
```

---

## Step 1 — Deploy the Backend on Render

### 1a. Create a Render account
Go to [render.com](https://render.com) and sign up (free tier works for demo).

### 1b. Connect the GitHub repo
1. Render dashboard → **New** → **Blueprint**
2. Connect `ksksrbiz-arch/nextchapter-travel-site`
3. Render detects `render.yaml` and creates `nextchapter-travel-api`

### 1c. Set environment variables
In Render → **nextchapter-travel-api** → **Environment**, add:

| Variable | Value | Where to find it |
|---|---|---|
| `DATABASE_URL` | `mysql://user:pass@host:3306/db` | Your MySQL host (see Step 2) |
| `VITE_APP_ID` | (your Manus app ID) | Manus project settings |
| `OWNER_OPEN_ID` | (your Manus open_id) | Manus profile |
| `BUILT_IN_FORGE_API_URL` | (Manus forge URL) | Manus project settings |
| `BUILT_IN_FORGE_API_KEY` | (Manus forge key) | Manus project settings |
| `RESEND_API_KEY` | `re_...` | [resend.com](https://resend.com) dashboard |
| `OAUTH_SERVER_URL` | `https://api.manus.im` | Fixed value |
| `JWT_SECRET` | (auto-generated by Render) | Leave as `generateValue: true` |

### 1d. Deploy
Click **Deploy** — Render builds the Docker image and starts the service.
Copy the service URL (e.g. `https://nextchapter-travel-api.onrender.com`).

---

## Step 2 — Set up a MySQL Database

**Option A — PlanetScale (recommended, free tier)**
1. [planetscale.com](https://planetscale.com) → New database → Region: US West
2. Create a branch `main` → Connect → copy the `DATABASE_URL`

**Option B — TiDB Cloud (serverless, free tier)**
1. [tidbcloud.com](https://tidbcloud.com) → Serverless cluster
2. Connect → copy the connection string

**Option C — Render MySQL add-on**
1. Render dashboard → **New** → **PostgreSQL** (or MySQL via add-on)
2. Copy the internal connection string

Paste the `DATABASE_URL` into the Render environment variable from Step 1c.

### Run database migrations
After the first deploy, run migrations via Render Shell:
```bash
# In Render → nextchapter-travel-api → Shell
node -e "require('./dist/index.js')"
# Or run drizzle migrations directly if you have a migration script
```

---

## Step 3 — Connect Netlify to the Backend

1. Netlify dashboard → **nextchapter-travel-site** → **Site configuration** → **Environment variables**
2. Add:

| Variable | Value |
|---|---|
| `BACKEND_URL` | `https://nextchapter-travel-api.onrender.com` |

3. Netlify → **Deploys** → **Trigger deploy** → **Deploy site**

The `netlify.toml` proxy rule rewrites all `/api/*` requests to your Render backend automatically.

| Check | URL |
|---|---|

| API health | `https://nextchapter-travel-api.onrender.com/api/trpc/auth.me` |

## Notes

- **Render free tier sleeps after 15 min of inactivity** — first request after sleep takes ~30s. Upgrade to Starter ($7/mo) for always-on.
- **SSE (real-time messaging)** requires the Render service to stay alive. Free tier sleep will disconnect active SSE clients.
- **S3 storage** (document vault, chat attachments) uses the Manus built-in S3 via `BUILT_IN_FORGE_API_KEY`. No separate AWS setup needed unless you want to use your own bucket.
