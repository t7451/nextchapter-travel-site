# Multi-Brand Asset Repository

This repository serves as a central hub for all reusable assets and materials across our multi-branded architecture.

## Structure

- `templates/`: Reusable templates (email, social media, web components, presentations)
- `documentation/`: Supporting documentation (guides, references, brand guidelines)
- `assets/`: Shared visual assets (logos, icons, images)
- `scripts/`: Automation scripts

## Directory Overview

### Templates (`templates/`)

Contains all reusable templates that can be adapted across different brands:

- **`email/`** - HTML email templates for various campaigns
  - Welcome emails
  - Transactional emails
  - Promotional campaigns
  
- **`social_media/`** - Social media post templates
  - Images and graphics
  - Caption templates
  - Platform-specific formats
  
- **`web/`** - Website components and partials
  - Reusable UI components
  - Landing page sections
  - Navigation patterns
  
- **`presentations/`** - Slide deck outlines and assets
  - Pitch decks
  - Training materials
  - Client presentations

### Documentation (`documentation/`)

Supporting documentation for implementing and using the assets:

- **`guides/`** - Implementation guides and best practices
  - Setup instructions
  - Integration guides
  - Best practices documentation
  
- **`references/`** - Technical references
  - Variable references
  - API documentation
  - Template specifications
  
- **`brand_guidelines/`** - Master brand guidelines
  - Visual identity standards
  - Voice and tone guidelines
  - Usage rules

### Assets (`assets/`)

Shared visual assets that maintain consistency across brands:

- **`logos/`** - Brand logos in various formats
  - SVG (preferred for web)
  - PNG (high-resolution)
  - PDF (print-ready)
  
- **`icons/`** - Consistent icon set
  - UI icons
  - Social media icons
  - Product icons
  
- **`images/`** - Stock photos and illustrations
  - Hero images
  - Background patterns
  - Product photography

### Scripts (`scripts/`)

Automation scripts for processing and managing assets:

- Build and optimization scripts
- Template generation tools
- Asset conversion utilities

## Contributing

### Adding New Assets

1. Place files in the appropriate directory based on type
2. Follow the naming conventions specified in the brand guidelines
3. Include a README in subdirectories if needed for context
4. Ensure all templates include clear variable placeholders (e.g., `{{ variable_name }}`)

### File Naming Conventions

- Use lowercase with hyphens: `my-template-name.html`
- Include version numbers where appropriate: `template-v2.html`
- Be descriptive: `welcome-email-day0.html` not `email1.html`

### Pull Request Guidelines

1. Ensure your additions follow the directory structure
2. Update relevant documentation
3. Test templates before submitting
4. Include screenshots or examples where applicable

## Getting Started

### For Developers

1. Clone this repository
2. Navigate to the appropriate directory for your needs
3. Copy templates to your project
4. Replace variable placeholders with actual content
5. Customize styling to match your brand

### For Designers

1. Access the `assets/` directory for logos and brand assets
2. Review `documentation/brand_guidelines/` for design standards
3. Use the templates in `templates/` as starting points
4. Maintain consistency with existing visual language

### For Marketing Teams

1. Browse `templates/email/` and `templates/social_media/` for ready-to-use templates
2. Refer to `documentation/guides/` for implementation instructions
3. Use variable references in `documentation/references/` to customize content
4. Submit feedback through pull requests or issues
